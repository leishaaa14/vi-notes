# Vi-Notes — Feature #4: Detect Pasted Text

This module implements paste detection for the [Vi-Notes](https://github.com/Vi-Sakha/Vi-Notes) project. It captures paste events in the writing editor, recording when a paste occurred, how much text was pasted, and a short preview — without storing the full content. This data is used to distinguish naturally typed content from pasted text during authenticity analysis.

## Available Scripts

In the project directory, you can run:

### `npm test`

Runs all unit tests for the paste detection module.\
Tests cover paste event recording, preview truncation, session counters, callback firing, listener cleanup, and summary generation.

All 11 tests should pass:

```
Tests: 11 passed, 11 total
```

### `npm run build`

Compiles the TypeScript source files to JavaScript.\
Output is placed in the `dist/` folder.

```
dist/
├── pasteDetector.js
└── usePasteDetector.js
```

The compiled `dist/pasteDetector.js` is what the demo and the broader Vi-Notes app import at runtime.

### `npx serve .`

Starts a local development server and opens the demo in your browser.\
Open [http://localhost:3000](http://localhost:3000) to view the demo.

The demo provides a distraction-free writing editor with live paste detection powered directly by `dist/pasteDetector.js`. Paste any text into the editor to see the event log, paste count, pasted character count, and typed authenticity score update in real time.

> **Note:** You must run `npm run build` before `npx serve .` to ensure `dist/pasteDetector.js` exists.

## Module API

### `createPasteDetector(element, options?)`

Attaches paste detection to a textarea or HTML element.

```ts
const detector = createPasteDetector(textareaEl, {
  onPaste: (event) => console.log(event),
  previewLength: 100  // default
});

// Remove listeners when done
detector.destroy();
```

Each `onPaste` callback receives a `PasteEvent` object:

```ts
{
  timestamp: string;           // ISO timestamp of the paste
  pastedLength: number;        // Number of characters pasted
  pastedPreview: string;       // First 100 chars (never full content)
  cursorPosition: number;      // Cursor position when paste occurred
  sessionPasteCount: number;   // Running total of paste events
  sessionPastedChars: number;  // Running total of pasted characters
}
```

### `buildPasteSummary(detector, totalEditorChars)`

Generates a summary suitable for saving to a writing session in MongoDB.

```ts
const summary = buildPasteSummary(detector, editor.value.length);
// {
//   totalPasteEvents: 2,
//   totalPastedChars: 340,
//   pasteEvents: [...],
//   pasteRatio: 0.6        // ratio of pasted chars to total chars
// }
```

### `usePasteDetector()` (React)

A React hook that manages the detector lifecycle automatically.

```tsx
const { ref, pasteEvents, getSummary } = usePasteDetector();

// Attach to your textarea
<textarea ref={ref} />

// On session save
const summary = getSummary(editorText.length);
```

## How It Fits Into Vi-Notes

This module implements [Feature Request #4](https://github.com/vicharanashala/vi-notes/tree/main) from the Vi-Notes project. The `PasteEvent` data it produces is designed to be saved to MongoDB alongside the keystroke timing data from Feature #3, giving the authenticity analysis engine a complete picture of how content was composed.

Paragraphs that appear in the editor without corresponding keystroke data, or sessions with a high `pasteRatio`, are flagged as potential AI-assisted content during report generation.
